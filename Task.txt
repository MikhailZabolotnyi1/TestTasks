Тестовое задание
Описание:
Вам необходимо автоматизировать три теста (описание тестов ниже). Третий тест необходимо автоматизировать в браузере. Желательно использовать для автоматизации браузер Chrome последней версии.

Подготовка окружения для выполнения тестов:
Создать SQLite базу данных(в виде файла), в ней создать таблицу Countries
Заполнить таблицу данными

	Population	Area
Ukraine	41588354	603628
France	67399000	640679
USA	328239523	9833520
China	1400050000	9596961
Сохранить изображение как example.jpeg по ссылке - http://apimeme.com/meme?meme=Alarm-Clock&top=Top+text&bottom=Bottom+text

TEST-1(API)
Через GET запрос получить изображение по ссылке http://apimeme.com/meme?meme=Alarm-Clock&top=Top+text&bottom=Bottom+text
Проверить, что изображение соответствует example.jpeg в стандарте base64

TEST-2(DB)
Используя данные из SQLite базы данных проверить:
плотность населения ниже 50 чел./кв.км. только у США. Плотность = Население / Площадь.
сумма население всех 4-х стран меньше 2 млрд человек

TEST-3 (iFrame)
Открыть в браузере https://www.w3schools.com/tags/tryit.asp?filename=tryhtml_iframe
В iframe source заменить на https://www.bing.com
Кликнуть на Run
В поле поиска вписать “Redmond”,дождаться выпадающего списка с рекомендованными запросами
Проверить, что в выпадающем списке есть строка “Redmond Washington”
Кликнуть по “Redmond Washington”
Проверить, что первая ссылка в результатах поиска https://www.bing.com/travelguide?q=Redmond



Требования:
Опубликовать на GitHub.
Использование тестового фреймворка (любого).
В описании на GitHub должно быть руководство к запуску вашего теста.
В описании на GitHub должно быть руководство как подготовить окружение(желательно если подготовка будет автоматически выполняться коммандой/скриптом).
Это должен быть именно тест, а не набор скриптов. Проект должен быть рабочим и запускаться. Для проверок используйте assert из различных библиотек (в зависимости от выбранного языка): JUnit, TestNG, Chai, PyTest и т.д.
Опционально (по желанию): Добавьте скриншоты в ваши тесты.
Опционально (по желанию): Добавьте возможность параллельного запуска тестов в вашем проекте (добавьте команду или инструкцию для параллельного запуска в описание проекта)
Тесты должны иметь возможность запускаться как индивидуально, так и все вместе
